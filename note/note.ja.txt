# coding: utf-8

概要
  基本的には勘でパースしています。

出力の読み方
  タイトル行には、それぞれ元データの記述箇所によってprefix(foo/)を付加しています。
  t: テンプレート単位でかかわって来るパラメータ。templateの略
  !: condition単位ではないパラメータ
  c: conditionの中心パラメータっぽいもの。coreの略
  s: SET 関連のパラメータ。setの略
  j: 元ファイルのどこにも出てこないが、参考資料として付加したもの。judgeの略

  このうち、各パラメータ内でも「t」prefixのみが登場する事があります。
  これは、「condition」の中で指定が無いけれど「template」の中で指定があるケースに対して「template」の値を記載した事を示しています。デフォルト値みたいなもんですね

Note
  厄介なポイント集。個人的なメモ
    * TEXTやHELPTEXTは複数行に渡る事が出来る。他のパラメータでもこういうものがあるかは不明
      その上、中に"を含む場合はエスケープ処理が効いて面倒。（今のとこ内部のエスケープ解釈は採り入れてない）
    * MSGCONDITIONS や SUPPRESSCONDITIONS という大枠は、一つのカテゴリつーかテンプレート内で何度でも現れる。
      おそらく、出現順が処理の優先度になっている。
      で、GUI上識別子との関連付けは多分こう
        +: MSGCONDITIONS
        =: SUPP_UNM_GCONDITIONS
        -: SUPPRESSCONDITIONS
  
次の2点は、特に不正確である可能性が高い。
どなたかが仕様を教えてくださるか、トラブったサンプルを提供して頂ければ対処できるかも・・・

  * 出現する順番
  * パラメータの網羅

とりあえず、どちらの想定外ケースへ直面しても例外を吐く仕様としておきます。
